# 复刻 1:1 设计规范

本文记录本次首页和菜谱页 1:1 复刻的执行方法，后续 agent 复刻其它页面时按同一流程走。目标是：用户不需要在聊天里反复发送截图，agent 也能从仓库里的设计源和规范继续复刻。

## 1. 基准输入

- 设计稿：`docs/ui-concepts/01-home.png` 到 `docs/ui-concepts/08-cook-review.png`
- 页面规格：`.agents/rules/style.md`
- 全局规范：`.agents/rules/global-style.md`
- 当前进度：根目录 `progress.md`
- 代码入口：
  - `src/components/import-flow.tsx`
  - `src/components/recipe-list.tsx`
  - `src/components/recipe-card.tsx`
  - `src/app/globals.css`
  - `src/components/bottom-nav.tsx`

截图原图为 `853 x 1844`，按 2x 左右导出理解，落地以 `430 x 932` 手机画布作为 1:1 基准。所有视觉判断先对齐 430px，再检查 393px、390px、375px。

如果后续 agent 不能读取 `docs/ui-concepts/*.png`，则不能直接声称能 1:1 复刻。此时必须先把目标页面补成“页面复刻包”，至少包括本规范第 12 节的字段，再开始实现。

## 2. 复刻流程

1. 先把截图拆成页面区域：
   顶部标题、全宽主图、主标语、导入操作行、最近做过列表、底部导航。
2. 把截图中的坐标和尺寸写入 `.agents/rules/style.md` 或本文件的页面复刻包，避免实现 agent 靠感觉猜。
3. 在 React 里保留业务流程，只替换首页 stage 的结构和 class hook。
4. 在 CSS 里用稳定类名写精确尺寸，不把关键布局散落在长 Tailwind class 串里。
5. 对真实视觉素材用本地静态资源，避免网络图加载失败或构图偏差。
6. 用单元测试锁住文案、结构、数据格式和关键 CSS 规则。
7. 用 Playwright 量取关键节点 bounding box，确认浏览器实际结果。

## 3. 首页结构实现

首页 stage 使用：

- 根节点：`.home-page`
- 顶部：`.home-header`
- 主图：`.home-hero-frame` + `.home-hero-image`
- 标语：`.home-promise`
- 导入行：`.home-import-row`
- 最近做过：`.home-recent`
- 底部导航：`.bottom-nav`

这些类是 1:1 复刻的稳定锚点。不要随手改名，否则测试和浏览器验收会失去定位点。

## 4. 全宽主图策略

`AppShell` 原本给页面加了左右 padding。首页截图里主图需要贴满 430px 容器，所以 `.home-page` 用负外边距抵消 shell padding：

```css
.home-page {
  --page-x: clamp(20px, 5.8vw, 25px);
  width: calc(100% + (var(--app-gutter) * 2));
  max-width: var(--app-max-width);
  margin: -20px calc(var(--app-gutter) * -1) 0;
}
```

主图本身保持：

```css
.home-hero-image {
  width: 100%;
  height: min(271px, 63vw);
  min-height: 236px;
  max-height: 271px;
  object-fit: cover;
  object-position: 58% 50%;
  border-radius: 10px;
}
```

## 5. 导入行策略

导入行必须占满内容横向宽度，箭头右对齐。用户在浏览器里标注过这个点。

关键规则：

```css
.home-import-row {
  display: grid;
  grid-template-columns: 48px minmax(0, 1fr) 32px;
  width: calc(100% - (var(--page-x) * 2));
  margin: 23px var(--page-x) 0;
}

.home-import-arrow {
  justify-self: end;
}
```

不要用内容宽度、`inline-flex` 或 `width: auto`。否则浏览器选中区域会只包住文字和箭头，看起来不是整行入口。

## 7. 数据格式

首页最近做过只显示两条，和设计稿一致：

```ts
recipes.filter((recipe) => recipe.cookedCount > 0).slice(0, 2)
```

元数据格式：

- 有 `cookTimeMinutes`：先显示 `45 分钟`
- 有 `wifeRating`：显示 `老婆评分 4.8`
- 无评分：显示 `做过 3 次`

因此列表 API 需要返回 `cookTimeMinutes`。

菜谱页重点首条元数据格式：

- 首条：`家常菜 · 45 分钟 · 做过 3 次 · 老婆评分 4.8`
- 普通行：`简单 · 30 分钟 · 做过 2 次`
- 未做过：`中等 · 25 分钟 · 还没做过`

## 10. 注意事项

- 不要把首页做成卡片墙。
- 不要恢复 `AppShell` padding 里的 hero 图。
- 不要用网络图替代本地设计稿裁剪图做 1:1 验收。
- 不要并行跑 `next dev` 和 `next build`，它们会同时写 `.next`，容易造成缓存 chunk 错误。
- 需要 build 时先停 dev server，再 `rm -rf .next` 后运行 build。

## 11. 无聊天截图交接规范

用户不在聊天里重新发送截图时，后续 agent 必须按以下顺序接手：

1. 完整阅读 `AGENTS.md`。
2. 完整阅读根目录 `progress.md`，从“下一步”继续。
3. 阅读 `.agents/rules/style.md`、`.agents/rules/global-style.md` 和本文。
4. 自己读取本地设计稿 `docs/ui-concepts/*.png`。不能要求用户重复上传同一批截图。
5. 如果当前运行环境无法读取 PNG，先停止实现，改为补齐目标页面的量化页面复刻包。
6. 实现前先写能失败的结构/文案/CSS 规则测试。
7. 实现后用浏览器量取关键 DOM 节点，不需要每轮都保存截图，但至少要确认无横向溢出、关键块尺寸、行高、首屏结构和底部固定栏。

交接提示词：

```text
先读 AGENTS.md 和根目录 progress.md。
当前是 1:1 复刻任务，规范在 .agents/rules/。
不要让我重新发截图，自己读取 docs/ui-concepts/*.png。
不能只靠 .agents/rules/style.md 猜 UI；必须对照本地设计稿或先补页面复刻包。
从 progress.md 的“下一步”继续，并保留当前未提交改动。
```

## 12. 页面复刻包字段

如果某个页面尚未像首页一样写到坐标级别，先为该页面补以下字段。缺任何一项，都只能算“相似实现”，不能叫 1:1。

- 设计稿文件：例如 `docs/ui-concepts/07-recipe-detail.png`
- 基准画布：原图像素、推导后的 CSS 画布宽高、状态栏是否计入
- 页面路由和代码入口：例如 `/recipes/[id]`、`src/components/recipe-detail.tsx`
- 根节点 class hook：例如 `.recipe-detail-page`
- 分区纵向坐标：每个大区的 top、height、margin-top、padding
- 横向边距：页面边距、图片是否全宽、内容是否抵消 `AppShell` padding
- 字体规格：标题、二级标题、正文、元信息、按钮文字的 font family、size、weight、line-height、color
- 图片规格：裁剪来源、输出路径、宽高、圆角、object-position
- 图标规格：lucide 图标名、尺寸、stroke-width、颜色、按钮触控区
- 列表/表单规格：行高、缩略图尺寸、分隔线位置、列宽、文本截断规则
- 固定区域：底部导航、固定按钮、弹层、安全区 padding、z-index
- 状态数据：为了预览 1:1，需要的 mock/临时 SQLite 数据
- 自动化验收：单元测试文件、DOM class 断言、浏览器 bounding box 或文本信号
- 禁止项：该页面不能出现的旧 UI 模式，例如毛玻璃卡片、渐变、营销式大 CTA

## 14. 接近 1:1 的验收阈值

- 430px 基准下，关键块横向位置误差不超过 2px。
- 430px 基准下，关键块高度误差不超过 4px。
- 375px、390px、393px 下不得横向溢出。
- 首屏关键文本不得换行错位、遮挡、被底部栏盖住。
- 图片必须来自本地裁图或明确的真实业务图片；不能用随机网络图代替设计稿构图。
- 用户指出单点视觉差异时，只改该差异相关结构/CSS，并补定向测试或浏览器量取记录。
