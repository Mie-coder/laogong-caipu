## 1. 设计方向

关键词：编辑感、克制、温暖、真实菜品、清晰工具性。

- 用真实菜品图片、字体层级、留白和发丝线建立高级感。
- 页面以暖白为主，不使用大面积彩色容器。
- 珊瑚色只用于选中、进度、评分和轻量提示。
- 深墨色用于标题、主按钮和关键操作。
- 页面区块默认不使用卡片；只有图片、抽屉和确实需要边界的控件可以使用圆角。
- 不使用全页面毛玻璃、渐变背景、装饰光斑或卡片套卡片。

## 2. 色彩

```css
--color-bg: #fff9f5;
--color-surface: #ffffff;
--color-ink: #2e2725;
--color-text: #3d3633;
--color-muted: #6f6865;
--color-subtle: #9a928e;
--color-line: #e9e3df;
--color-accent: #ff6b6b;
--color-accent-soft: #fff0ed;
--color-overlay: rgba(24, 20, 18, 0.52);
--color-disabled: #c9c2be;
```

使用比例：

- 暖白与白色：约 80%。
- 深墨色与正文色：约 15%。
- 珊瑚色：不超过 5%。

禁止：

- 珊瑚色大面积铺底。
- 紫色、蓝紫渐变。
- 米色或棕色主导整个页面。
- 用不同背景色区分每一个内容区块。

## 3. 字体

字体栈：

```css
font-family:
  "PingFang SC",
  "Hiragino Sans GB",
  "Microsoft YaHei",
  system-ui,
  sans-serif;
```

字号：

| 用途 | 字号 | 字重 | 行高 |
| --- | ---: | ---: | ---: |
| 品牌标题 | 36px | 700 | 1.2 |
| 页面标题 | 30px | 700 | 1.25 |
| 菜谱标题 | 28px | 700 | 1.3 |
| 区块标题 | 20px | 600 | 1.4 |
| 正文强调 | 17px | 600 | 1.5 |
| 正文 | 16px | 400 | 1.65 |
| 辅助文字 | 14px | 400 | 1.5 |
| 元数据 | 13px | 400 | 1.45 |
| 微型提示 | 12px | 400 | 1.4 |

规则：

- 字间距固定为 `0`。
- 标题不使用渐变、描边或阴影。
- 同一区域最多使用三个文字层级。
- 长步骤必须自然换行，不截断主要内容。
- 数字步骤可使用更大的斜体或衬线感字形，但正文仍使用无衬线字体。

## 4. 间距与布局

- 设计基准宽度：390px。
- 支持宽度：375px 至 430px。
- 页面最大宽度：430px，水平居中。
- 页面左右安全间距：20px。
- 主要区块垂直间距：32px。
- 区块内部间距：16px。
- 紧凑元素间距：8px。
- 固定导航和操作栏必须包含底部安全区。

间距只使用：

```text
4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48
```

## 5. 圆角、边框与阴影

圆角：

- 图片：6px。
- 输入框、主按钮：8px。
- 底部抽屉顶部：8px。
- 缩略图：6px。
- 禁止 16px 以上的大圆角卡片。
- 禁止胶囊形按钮，除非是极小的状态标签。

边框：

- 默认发丝线：`1px solid #e9e3df`。
- 输入聚焦：`1px solid #2e2725`。
- 选中状态：`1px solid #ff6b6b`。
- 页面分区优先使用顶部或底部单线，不使用完整矩形边框。

阴影：

- 普通页面元素不使用阴影。
- 底部抽屉允许：`0 -8px 30px rgba(46, 39, 37, 0.10)`。
- 悬浮层允许极轻阴影，不使用发光效果。

## 6. 图片

- 菜品图片必须明亮、真实、可辨认，避免暗调和过度裁切。
- 首页主图接近全宽，承担首屏视觉重心。
- 图片使用 `object-fit: cover`，但不能裁掉主要菜品。
- 列表缩略图保持统一比例和尺寸。
- 无图时使用安静的中性占位，不使用大尺寸 emoji。
- 图片上只允许必要的“封面”和全屏查看控件。

建议比例：

- 首页主图：`4 / 3`。
- 列表重点菜谱：`16 / 9`。
- 列表缩略图：`4 / 3`。
- 图片审核主图：接近 `1 / 1`。

## 7. 按钮

### 主按钮

- 背景：`#2e2725`。
- 文字：白色。
- 高度：48px。
- 圆角：8px。
- 不使用渐变、毛玻璃或大面积珊瑚色。
- 按下时透明度降至 88%，不缩放页面布局。

### 次按钮

- 透明背景。
- 深墨色文字或图标。
- 使用发丝线或纯文字表达层级。

### 图标按钮

- 使用 Lucide 线性图标。
- 图标尺寸：20–24px。
- 最小点击区域：44px。
- 无必要时不添加圆形背景。
- 必须有可访问名称。

## 8. 输入控件

- 输入框背景以白色或透明为主。
- 只使用细边框或底部线。
- 不使用浮雕、毛玻璃和粗描边。
- 标签位于输入框上方。
- 错误信息紧邻字段，以小珊瑚色圆点加文字表达。
- 多行输入框根据内容伸展，初始高度保持克制。

## 9. 标签与筛选

- 标签使用透明背景和 `1px` 细边框。
- 圆角不超过 4px。
- 选中状态使用珊瑚色描边，不使用整块填充。
- 页面级标签切换使用文字加下划线，不使用分段胶囊。
- 分类作为菜谱列表筛选项，不进入底部导航。

## 10. 导航

底部导航只保留：

1. 导入
2. 菜谱

规则：

- 白色背景。
- 顶部 `1px` 发丝线。
- 不使用毛玻璃大色块。
- 当前项通过珊瑚色图标、文字和短下划线表示。
- 导航高度稳定，内容不得被遮挡。

顶部导航：

- 返回、历史、编辑、更多使用线性图标。
- 标题居中或与页面内容左对齐。
- 不使用独立彩色导航栏。

## 11. 底部抽屉

- 白色实体背景。
- 顶部圆角 8px。
- 遮罩为深色半透明，不重度模糊背景。
- 顶部包含拖动条、标题和关闭按钮。
- 内容区允许滚动，主要操作固定在安全区上方。
- 抽屉动画：从底部进入，时长 240–300ms。

## 12. 动效

- 页面进入：轻微淡入和 8px 上移，180–240ms。
- 抽屉：底部滑入，240–300ms。
- 选中：颜色和边框过渡，120–180ms。
- 列表删除：淡出并收起高度，200–260ms。
- 解析状态使用小型进度环，不使用整屏动画。
- 尊重 `prefers-reduced-motion`。

禁止：

- 弹跳过强的 spring。
- 点击后大幅缩放。
- 长时间循环装饰动画。
- 全屏彩纸庆祝。

## 13. 状态反馈

- Toast：白色或深墨色实体背景，显示约 2 秒。
- 错误：字段附近说明，不清空用户输入。
- 危险操作：使用确认对话框或底部抽屉。
- 加载：使用与真实布局一致的骨架，不使用独立大卡片。
- 空状态：简短文案和一个明确行动，不使用夸张插画。

## 14. 禁止项

- 全局渐变背景。
- 大面积珊瑚色按钮和面板。
- 全页面毛玻璃。
- 卡片套卡片。
- 圆角大于 16px。
- 胶囊式导航、筛选和主按钮。
- 大量 emoji 代替图标。
- 为每个区块添加背景、边框和阴影。
- 通过装饰而不是信息层级制造“高级感”。
