## 9. 失败退回与重验记录

任何必过项失败时，按以下流程执行：

1. 将失败项标记为“失败”，保存命令输出、截图或复现步骤。
2. 当前 Task 不得标记完成，`progress.md` 不得写成通过。
3. 将失败证据、对应规格和允许修改的文件范围交给实现 agent 或专用修正 agent。
4. 修正 agent 必须运行覆盖该问题的定向测试并记录命令、结果和 commit。
5. 独立 reviewer 对修正 diff 重新进行规格与质量复审。
6. 主控重新执行失败关卡；如修正可能影响后续关卡，从受影响的最早关卡开始重跑。
7. 只有复审和主控重验均通过后，失败项才能改为“通过”。
8. 未经用户明确接受，不得把失败项降级成“已知差异”后宣布整体通过。
