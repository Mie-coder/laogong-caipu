# 老公菜谱移动端视觉与动态食材图修订设计

日期：2026-07-19

状态：待书面复核（用户已选择方案 B：做菜模式懒生成并缓存）

## 1. 目标

修正真实手机试用中发现的视觉密度、排版、对齐和做菜交互问题，并为做菜模式增加安全、可缓存的动态食材生图能力。

本轮只处理以下范围：

- 图片审核页封面标识居中和标题层级。
- 解析页标题密度及步骤轴线对齐。
- 确认菜谱页标题、时间和标签布局。
- 做菜复盘抽屉的分隔层次。
- 做菜模式底部按钮组密度。
- 做菜步骤整行点击完成、删除线状态和进度计数。
- 做菜模式食材卡片的 Micu 动态图片生成与缓存。
- 全站中文字体和字号层级重新统一。

不重写菜谱解析、数据库主模型、计时器、语音播报或其他已验收功能。

## 2. 已确认方案

动态食材图采用方案 B：

1. 做菜模式先立即显示轻量占位，不等待图片生成。
2. 食材卡片进入横向轨道可视区域时，客户端请求服务端生成图片。
3. 图片完成后逐张淡入替换，占位期间不阻塞步骤、计时或语音。
4. 结果按规范化食材名称和提示词版本缓存，相同食材跨菜谱复用。
5. 失败时保留占位；重新进入或手动重试时再次尝试。

不采用导入阶段同步生成，因为会延长原本数秒的解析流程；不引入后台任务队列，因为当前单用户本地应用没有相应基础设施。

## 3. 排版系统

### 3.1 字体

- 中文主字体使用自托管的思源宋体 `Noto Serif SC`，不依赖运行时字体 CDN。
- 备用顺序为 `Songti SC`、`STSong`、系统中文衬线字体和 `serif`。
- `button`、`input`、`textarea` 和 `select` 继续继承全局字体。
- 数字计时使用 `font-variant-numeric: tabular-nums`，避免倒计时宽度跳动。
- 使用 `font-display: swap`，字体加载失败不得造成不可操作或空白页面。

### 3.2 字号层级

| 语义 | 字号 | 行高 | 使用位置 |
| --- | ---: | ---: | --- |
| Display | 32px | 1.15 | 仅首页、菜谱列表页主标题 |
| Page title | 24px | 1.3 | 图片审核、解析、确认菜谱等独立页面 |
| Navigation title | 18px | 1.35 | 做菜模式等紧凑顶部导航 |
| Dish title | 24px | 1.3 | 确认页、详情页菜名 |
| Section title | 20px | 1.4 | 食材、步骤、复盘区块 |
| Body | 16px | 1.65 | 步骤和主要正文 |
| Label | 14px | 1.45 | 元数据、按钮和辅助标签 |
| Chip | 13px | 1.3 | 菜谱标签、筛选胶囊 |

除首页和菜谱列表主标题外，不允许出现 28px 以上的页面标题。标题主要通过字重、留白和颜色建立层级，不再依靠不断放大字号。

## 4. 六项界面修订

### 4.1 图片审核

- 封面标识使用统一的 shadcn `Button`/语义标签外观。
- `封面图` 和 `设为封面` 容器都使用 `inline-flex`、双轴居中和固定行高，避免 `span` 与 Button 的基线差异。
- 页面标题固定为 24px，副标题 14px。
- 底部操作继续使用 shadcn `Button`，保留安全区和 44px 以上触控高度。

### 4.2 解析进度

- 页面标题由当前大标题收敛为 24px。
- 步骤标题为 16px，说明为 13px。
- 步骤轴使用固定 36px 标记轨道、32px 状态圆点；完成、当前、等待三种状态不得改变外层轨道宽度。
- 连接线固定穿过圆点中心，状态切换不允许产生横向跳动。
- 长分享文本在来源卡片内正常换行，不撑破页面。

### 4.3 确认菜谱

- 页面标题 24px，菜名 24px，区块标题 20px。
- 封面缩至 88px，给菜名、标签和时间留出更多横向空间。
- 标签使用 13px、紧凑内边距和不低于 32px 的点击高度。
- 时间控件作为不换行的紧凑组：时钟、数字输入和“分钟”始终在同一行。
- 数字输入按字符宽度布局，不再使用会挤压“分钟”的固定窄列。

### 4.4 做菜复盘抽屉

- 删除“做菜时间”上方及 Drawer footer 上方的贯穿式实线。
- 用留白、轻微表面色差和圆角承载做菜时间，不使用双重边框分组。
- 底部保存按钮继续使用 shadcn `Button`，安全区不变。

### 4.5 做菜底部按钮组

- 375px、390px、430px 宽度下按钮组必须保持单行且无水平溢出。
- 减少、增加、停止和开始/暂停的触控目标保持至少 44px。
- 计时胶囊约 126px；两个圆形动作按钮各 44px；完成按钮使用紧凑标签和最小必要内边距。
- 组间距使用 6px 左右的稳定节奏，禁止依靠负 margin 强行塞入。
- 固定栏保留半透明材质和安全区；减少透明度偏好下退化为实色。

### 4.6 做菜步骤

- 移除每行右侧 Checkbox，不再要求点击小勾选框。
- 整行基于 shadcn `Button` 的 ghost 变体，点击即切换完成状态。
- 完成后步骤正文显示贯穿文字的删除线，整体降低对比度；步骤编号同步弱化。
- 再次点击可撤销，符合防误触和可恢复原则。
- 点击完成当前步骤后，将当前步骤推进到下一条未完成步骤；点击任意其他步骤时仍正确更新进度。
- 右上角计数由完成集合推导，显示 `1 / 8`，不得维护第二份独立计数。
- 完成全部步骤后才启用“完成做菜”，但不自动打开复盘抽屉。

## 5. 动态食材图片

### 5.1 服务与参数

- 服务：Micu OpenAI-compatible image relay。
- Endpoint：`https://www.micuapi.ai/v1/images/generations`。
- Model：`gpt-image-2`。
- Size：`1024x1024`。
- Quality：`low`，因为最终展示尺寸约 72px；如实测质量不足再提升为 `medium`。
- 请求使用 HTTP/1.1，并提供 Micu 兼容的 User-Agent。
- 同时兼容 `{ data: [{ url }] }` 与 `{ data: [{ b64_json }] }` 返回。

图片提示词由服务端固定生成，不接受客户端任意 prompt：

```text
中国家常菜食材摄影，单一食材“{食材名}”，俯拍居中，真实自然纹理，
暖米白宣纸色背景，柔和自然光，适合圆形裁切，无文字、无包装、无餐具、无水印。
```

### 5.2 服务端边界

- 密钥只从服务端进程环境变量 `MICU_API_KEY` 读取。
- 密钥不得写入 `.env`、源码、数据库、日志、测试快照或客户端响应。
- API 接收 `recipeId + itemKind + itemIndex`，服务端从已保存菜谱中反查食材名称；客户端不能提交自由文本 prompt。
- 生成接口使用 Zod 校验并限制到实际存在的食材或调料。
- 外部调用设置超时、响应大小上限和可理解的错误信息。

### 5.3 缓存

- 缓存目录：`data/generated/ingredients/v1/`，该目录已位于 Git 忽略范围。
- 文件键：`sha256("v1|" + normalizedIngredientName)`，避免路径穿越并支持跨菜谱复用。
- 写入先落临时文件再原子替换，避免并发读取半张图片。
- 同一进程内对相同 key 共享生成 Promise，避免三次并发请求重复扣用量。
- 图片通过受控 API route 返回，带稳定的 MIME、ETag/缓存头；不把运行时文件写进 `public/`。

### 5.4 客户端行为

- `IngredientRail` 初始渲染不等待网络。
- 使用 `IntersectionObserver` 观察横向轨道卡片，只为即将进入可视区域的食材请求图片。
- 加载中显示米白色骨架和食材首字；生成成功使用短交叉淡入。
- 网络或生成失败时保留首字占位和可访问名称，做菜流程保持可用。
- 组件卸载时取消未完成的客户端请求；服务端已开始的同名生成仍可完成并写入缓存。

## 6. Apple Design 交互约束

- 按压反馈从按下开始，缩放约 `0.98`，松开立即恢复。
- 步骤完成状态即时更新，不等待动画结束。
- 非手势加载图片使用短交叉淡入，不添加弹跳。
- 删除线、颜色和透明度变化控制在约 160–200ms，保持克制。
- `prefers-reduced-motion` 下取消缩放和位移动画，仅保留即时状态变化或极短淡入。
- 所有生成、动画和语音失败都不得阻断核心做菜流程。

## 8. 错误与恢复

- 未配置 `MICU_API_KEY`：返回明确的服务端配置错误，客户端保留占位。
- Micu 401/403：不重试，记录不含密钥的状态码。
- 429/5xx/超时：本次显示占位，下次进入可再次请求；不做无限自动重试。
- URL 下载失败或 base64 非法：不写缓存文件。
- 缓存文件损坏：删除该文件并在下一次请求重新生成。
- 字体加载失败：回退到系统宋体栈，不影响布局和操作。

## 10. 验收标准

- 图片审核封面文字视觉居中。
- 除首页和菜谱列表外，无超大页面标题；字体呈现统一的中文宋体气质。
- 解析步骤圆点、连接线和文字在所有状态下保持同一轴线。
- 确认页 `15 分钟` 不换行，标签尺寸克制。
- 复盘抽屉不再出现贯穿底部的实线。
- 375px 做菜底栏保持单行，每个主要操作仍可可靠点击。
- 做菜步骤没有 Checkbox；点击整行后出现删除线并更新 `x / n`，再次点击可撤销。
- 食材图片在后台动态生成、逐张出现并在相同食材再次使用时命中缓存。
- 密钥不进入 Git diff、浏览器 bundle、网络响应或日志。
- Micu 不可用时，做菜核心流程仍完整可用。
