# 老公菜谱 v0.1 项目现状归档

归档日期：2026-07-02

代码版本：`0.1.0`

归档基线：Git `86d6c36`

## 1. 项目概述

“老公菜谱”是一个本地单人使用的手机 H5 菜谱工具。当前版本围绕两件事展开：

1. 把小红书分享文本抓取并整理成结构化菜谱。
2. 保存每次做菜后的老婆评价和老公改进事项。

产品当前是可运行的 MVP，不包含账号、多用户同步和正式线上运营能力。

## 2. 当前产品形态

### 2.1 主要页面

| 页面 | 路径 | 当前状态 | 主要用途 |
| --- | --- | --- | --- |
| 菜谱导入 | `/` | 已实现 | 粘贴小红书分享文本、解析、审核图片、确认并保存 |
| 菜谱列表 | `/recipes` | 已实现 | 搜索、难度筛选、浏览和批量删除菜谱 |
| 菜谱详情 | `/recipes/[id]` | 已实现 | 查看图片、食材、步骤、小贴士和最近复盘 |
| 分类页面 | `/categories` | 部分实现 | 当前仅复用了菜谱列表，尚无分类入口和分类聚合界面 |

底部导航目前只有“导入”和“菜谱”两个入口。

### 2.2 核心用户流程

```text
粘贴分享文本
  → 识别小红书短链
  → 展开链接并抓取页面
  → AI/Mock 生成结构化菜谱
  → AI/规则筛选图片
  → 用户删除不合适的图片
  → 确认菜名、分类、标签和小贴士
  → 保存到 SQLite
  → 在菜谱列表查看
  → 打开详情并按步骤做菜
  → 滑动到底部
  → 记录老婆评分、评价和老公改进事项
```

## 3. 已实现功能

### 3.1 菜谱导入

- 支持粘贴包含 `xhslink.com` 的小红书分享文本。
- 自动提取分享文案、平台和短链接。
- 自动跟随重定向并抓取标题、描述、正文和图片。
- 抓取失败时仍可使用分享文案继续解析。
- 内容不足时提示用户补充原文并重新解析。
- 支持 `mock` 和 `deepseek` 两种 AI Provider。
- AI 输出经 Zod 校验和标准化后才能进入保存流程。
- 自动清理噪声标签、补齐食材结构、排序步骤并推断分类标签。

### 3.2 图片处理

- 从抓取页面中提取多张图片。
- DeepSeek 可筛除头像、表情包、推广图和无关图片。
- 未配置 DeepSeek 时使用 URL 规则进行基础过滤。
- 用户可在保存前逐张删除图片。
- 第一张保留图片作为封面。
- 菜谱详情保留并展示全部选中图片。
- 图片轮播支持前后切换、页码、指示点和全屏查看。

### 3.3 菜谱确认与保存

- 可修改菜名、主分类、标签和小贴士。
- 可查看 AI 生成的难度、食材、调料和步骤。
- 食材按肉类、禽蛋、水产、蔬菜、豆制品、主食、调料和其他分组展示。
- 保存成功后跳转菜谱列表，并高亮新保存的菜谱。
- 菜谱及关联图片、食材、步骤和标签通过事务写入 SQLite。

当前确认页不能直接增删或修改食材、用量和步骤。

### 3.4 菜谱列表

- 按更新时间倒序展示菜谱。
- 支持按菜名实时搜索。
- 支持按简单、中等、困难筛选。
- 卡片展示封面、分类、菜名、难度、标签、做过次数和最近评分。
- 长按 500ms 进入批量删除模式。
- 删除菜谱时级联删除图片、食材、步骤、标签和做菜记录。
- 加载时展示骨架屏；无数据时展示空状态。

接口已支持按分类和标签筛选，但当前页面没有完整的筛选入口。

### 3.5 菜谱详情与做菜复盘

- 展示菜名、分类、难度、做过次数和全部菜谱图片。
- 展示分组后的食材、调料、步骤和小贴士。
- 页面接近底部时显示“标记做过”按钮。
- 支持 1 至 5 级老婆评分。
- 支持老婆文字评价和老公改进事项。
- 保存复盘后增加做过次数并播放庆祝动画。
- 详情页展示最近一次复盘。

数据库会保存全部复盘记录，但当前界面只展示最近一条。

## 4. 后端与数据能力

### 4.1 API

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `POST` | `/api/import/parse` | 解析导入文本并返回菜谱草稿和图片 |
| `POST` | `/api/images/filter` | 筛选适合作为菜谱配图的图片 |
| `GET` | `/api/recipes` | 查询菜谱列表 |
| `POST` | `/api/recipes` | 保存菜谱 |
| `GET` | `/api/recipes/[id]` | 获取菜谱详情 |
| `DELETE` | `/api/recipes/[id]` | 删除菜谱 |
| `POST` | `/api/recipes/[id]/cook` | 新增做菜复盘 |

### 4.2 数据表

| 数据表 | 用途 |
| --- | --- |
| `recipes` | 菜谱主信息、分类、封面、难度和做过次数 |
| `recipe_images` | 菜谱多图 |
| `recipe_ingredients` | 食材与调料 |
| `recipe_steps` | 制作步骤 |
| `recipe_tags` | 标签 |
| `cooking_logs` | 做菜时间、评分、评价和改进事项 |
| `imports` | 导入过程预留记录表，当前业务流程尚未写入 |

## 7. 当前边界与已知不足

### 7.1 产品边界

- 仅支持本地单人使用。
- 主要导入来源只有小红书分享文本。
- 不支持登录、家庭成员、多设备同步或云端备份。
- 不支持购物清单、周菜单、库存或营养分析。
- 不处理复杂反爬、验证码和登录态抓取。

### 7.2 已实现但体验不完整

- 分类页没有分类卡片、分类统计和明确导航入口。
- 标签、分类筛选已有接口能力，但前端入口不完整。
- 确认页只能编辑部分字段，不能编辑食材和步骤。
- 详情页不能编辑菜谱。
- 全部复盘已保存，但只显示最近一次。
- `notes` 字段存在于接口和数据库，复盘表单没有输入入口。
- 导入记录表已建表，但当前没有写入或查看导入历史。
- 批量删除逐条请求，适合当前小数据量。

### 7.3 技术限制

- 小红书页面内容可能因登录限制或反爬策略抓取失败。
- AI 输出质量依赖抓取内容和模型配置。
- 图片使用外链，源站失效后图片可能无法显示。
- SQLite 适合本地单机，不适合直接作为多实例线上数据库。
- 本地原生模块需要匹配 Node ABI；重新安装依赖时需注意 `better-sqlite3` 版本。

## 8. 当前完成度判断

| 模块 | 完成度 | 判断 |
| --- | --- | --- |
| 导入主流程 | 85% | 可完整走通，异常恢复和字段编辑仍可加强 |
| 图片审核 | 80% | 已支持筛选、删除、轮播和全屏 |
| 菜谱管理 | 70% | 浏览、搜索、难度筛选、删除已完成，编辑和分类不足 |
| 做菜复盘 | 75% | 可记录评分与改进，只缺历史管理和更完整字段 |

| 多用户与同步 | 0% | 不在 v0.1 范围 |

总体判断：当前版本已经形成“导入 → 保存 → 做菜 → 复盘”的闭环，适合本地试用和验证产品方向；下一版应优先优化信息架构、菜谱编辑、分类浏览和复盘历史。
