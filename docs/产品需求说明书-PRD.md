# 老公菜谱产品需求说明书

版本：v0.2 原型优化输入稿

编写日期：2026-07-02

基线：v0.1 当前实现

## 1. 产品定义

### 1.1 产品名称

老公菜谱

### 1.2 一句话定位

把散落在小红书里的菜谱快速整理成可执行步骤，并持续记录每次做菜效果的家庭菜谱工具。

### 1.3 产品目标

1. 降低收藏菜谱后无法真正照着做的整理成本。
2. 让用户在做菜时快速找到食材、用量和步骤。
3. 记录老婆反馈和下次改进事项，让同一道菜越做越好。

### 1.4 目标用户

- 主要用户：负责找菜谱、做菜和改进的“老公”。
- 关联用户：负责品尝、评分和给反馈的“老婆”。
- 使用环境：家庭、手机端、单人维护、低频录入、高频查看。

### 1.5 核心价值

- 导入快：复制分享文本即可形成结构化菜谱。
- 做菜清楚：食材、用量、步骤和图片集中展示。
- 能复盘：每次评分、评价和改进事项与菜谱长期绑定。

## 2. 版本范围

### 2.1 v0.1 已实现

- 小红书分享文本导入。
- 网页抓取和 AI 菜谱解析。
- 图片自动筛选和人工删除。
- 菜谱确认、保存、搜索、难度筛选和批量删除。
- 菜谱详情和图片全屏查看。
- 做菜次数、老婆评分、评价和改进事项。

### 2.2 v0.2 原型优化重点

- 重新梳理底部导航和页面层级。
- 完善菜谱确认编辑能力。
- 建立真正可用的分类浏览。
- 增加菜谱编辑和复盘历史。
- 优化做菜场景下的详情阅读效率。

### 2.3 本版不包含

- 登录和账号体系。
- 社区、关注、评论和公开分享。
- 电商购买、外卖或食材配送。
- 营养计算、卡路里管理。
- 多人实时协作。

## 3. 信息架构

建议下一版采用 3 个底部导航：

| 导航 | 作用 | 默认页面 |
| --- | --- | --- |
| 导入 | 新增菜谱 | 导入页 |
| 菜谱 | 搜索和浏览全部菜谱 | 菜谱列表 |
| 分类 | 按分类快速找菜 | 分类首页 |

二级页面：

- 菜谱确认页
- 菜谱详情页
- 菜谱编辑页
- 分类结果页
- 复盘填写页或底部抽屉
- 复盘历史页

## 4. 核心用户流程

### 4.1 导入菜谱

```text
进入导入页
→ 粘贴小红书分享文本
→ 点击开始解析
→ 查看解析进度
→ 审核候选图片
→ 检查并编辑结构化菜谱
→ 保存
→ 查看新菜谱详情
```

异常分支：

- 未识别到链接：使用分享文字继续解析，并提示可补充正文。
- 抓取失败：保留已输入内容，允许手动补充后重试。
- AI 解析失败：展示明确错误，不清空用户输入。
- 内容不完整：高亮缺失字段，允许用户手动补齐。
- 图片全部删除：允许无图保存，使用默认占位图。

### 4.2 查找并做菜

```text
进入菜谱列表或分类
→ 搜索/筛选菜谱
→ 打开详情
→ 查看备料清单
→ 按步骤做菜
→ 滑到底部
→ 标记做过
```

### 4.3 做菜复盘

```text
选择老婆评分
→ 填写老婆评价
→ 填写老公下次改进
→ 保存
→ 做过次数 +1
→ 详情页展示最新复盘
→ 可进入历史查看全部记录
```

## 5. 页面需求

### 5.1 导入页

#### 页面目标

让用户用最少操作开始一次菜谱导入。

#### 页面内容

- 页面标题“老公菜谱”。
- 分享文本输入框。
- 粘贴按钮。
- “开始解析”主按钮。
- 最近导入失败时可保留输入内容。

#### 交互规则

- 输入为空时主按钮禁用。
- 点击粘贴后读取剪贴板内容；读取失败时聚焦输入框。
- 解析过程中按钮显示当前阶段：
  - 正在识别链接
  - 正在抓取内容
  - 正在整理菜谱
  - 正在筛选图片
- 解析中禁止重复提交。
- 出错后输入内容不得丢失。

#### 原型优化建议

- 将大段示例文案改为简短 placeholder 和单独的示例入口。
- 用四阶段进度反馈替代按钮文字连续变化。
- 增加“仅粘贴正文也可解析”的弱提示。

### 5.2 图片审核页

#### 页面目标

快速留下真正有助于做菜的图片。

#### 页面内容

- 大图预览。
- 图片数量和当前位置。
- 缩略图列表。
- 删除当前图片按钮。
- “确认图片”主按钮。
- “全部取消选择”或无图保存入口。

#### 交互规则

- 默认选中 AI 推荐图片。
- 点击缩略图切换大图。
- 删除后自动定位到相邻图片。
- 至少保留一张时，第一张为默认封面。
- 支持点击大图进入全屏查看。

#### 原型优化建议

- 使用可勾选缩略图，比逐张删除更适合多图审核。
- 提供“设为封面”，不再把第一张隐式当作封面。
- 标注“AI 推荐”而不是“AI 已筛选”，避免用户误认为结果绝对正确。

### 5.3 菜谱确认页

#### 页面目标

在保存前快速修正 AI 结果。

#### 可编辑字段

- 菜名，必填。
- 主分类，必填。
- 标签，最多 5 个。
- 难度：简单、中等、困难、未知。
- 预计用时。
- 食材名称、用量和类型。
- 调料名称和用量。
- 步骤顺序和步骤内容。
- 小贴士。
- 封面和菜谱图片。

#### 交互规则

- 食材和步骤支持新增、删除、修改和排序。
- 缺失或低置信度字段需显著提示。
- 菜名为空或步骤为空时不能保存。
- 离开页面前如有未保存修改，需二次确认。
- 保存期间禁止重复点击。

#### 原型优化建议

- 默认展示“简洁检查模式”，需要时再展开编辑。
- 食材与步骤分别使用紧凑列表，不使用大卡片嵌套。
- 标签改为可选标签加自定义输入，降低格式错误。

### 5.4 菜谱列表页

#### 页面目标

快速找到想做的菜。

#### 页面内容

- 页面标题和菜谱总数。
- 搜索框。
- 筛选入口。
- 菜谱卡片列表。
- 空状态。
- 批量管理模式。

#### 菜谱卡片

- 封面。
- 菜名。
- 主分类。
- 难度。
- 预计用时。
- 最多 3 个标签。
- 做过次数。
- 最近老婆评分。

#### 筛选项

- 主分类。
- 标签。
- 难度。
- 是否做过。
- 最近新增或最近做过。

#### 交互规则

- 点击卡片进入详情。
- 搜索按菜名匹配。
- 筛选条件可组合。
- 已生效筛选需显示为可取消条件。
- 长按进入批量管理可保留，但应同时提供可发现的“管理”入口。
- 删除前显示菜名和数量，并要求确认。

#### 原型优化建议

- 将难度筛选整合到统一筛选抽屉。
- 增加排序，不把所有条件挤在列表顶部。
- 卡片优先显示做菜决策信息，减少装饰元素。

### 5.5 分类页

#### 页面目标

在不知道具体菜名时，按类型发现已有菜谱。

#### 页面内容

- 分类网格或列表。
- 每个分类的名称、代表图片和菜谱数量。
- “未分类”入口。
- 常用标签区。

#### 建议分类

- 家常菜
- 快手菜
- 下饭菜
- 汤羹
- 凉菜
- 面食
- 烘焙
- 甜品
- 饮品
- 其他

#### 交互规则

- 点击分类进入分类结果页。
- 分类结果页复用菜谱列表和筛选能力。
- 无菜谱分类不展示，或置于末尾。
- 分类名称以菜谱保存数据为准，不创建重复同义词。

### 5.6 菜谱详情页

#### 页面目标

让用户在备料和做菜过程中不需要反复查找信息。

#### 页面结构

1. 顶部标题、分类、难度、用时和做过次数。
2. 图片轮播。
3. 食材与调料。
4. 制作步骤。
5. 小贴士。
6. 最近复盘。
7. 固定或阶段性出现的“标记做过”按钮。

#### 交互规则

- 图片支持全屏查看和左右切换。
- 食材按类型分组。
- 步骤序号清晰，并支持长文本换行。
- “标记做过”应在阅读完步骤后保持可发现。
- 提供编辑和删除入口。
- 删除需二次确认。
- 点击复盘区域进入全部复盘历史。

#### 原型优化建议

- 增加“备料 / 步骤”分段切换或锚点导航。
- 做菜模式可放大字号、保持屏幕常亮，并逐步勾选步骤。
- 不建议完全隐藏“标记做过”，可使用底部弱化悬浮按钮。

### 5.7 菜谱编辑页

#### 页面目标

修正已保存菜谱，不必重新导入。

#### 功能要求

- 编辑范围与确认页一致。
- 保存后更新菜谱详情。
- 支持增删图片和更换封面。
- 保留来源链接和原始分享文本。
- 不影响已有复盘记录。

### 5.8 做菜复盘

#### 页面目标

用低成本记录这次做菜结果和下次改进。

#### 输入字段

- 老婆评分：1 至 5，允许暂不评分。
- 老婆评价：选填。
- 老公改进事项：选填。
- 其他备注：选填。
- 做菜时间：默认当前时间，可修改。

#### 交互规则

- 至少填写评分或一项文字内容后才能保存。
- 保存后做过次数增加 1。
- 保存成功展示简短庆祝反馈。
- 再次打开表单时清空上次输入。
- 历史记录按时间倒序展示。

#### 原型优化建议

- 评分使用星级和文字说明，减少表情含义歧义。
- 增加常用改进标签，如“少盐”“火小一点”“时间短一点”。
- 历史记录支持编辑和删除，但可放到后续版本。

## 6. 状态与反馈

所有关键页面需覆盖以下状态：

- 初始状态。
- 加载状态。
- 成功状态。
- 空状态。
- 部分数据缺失状态。
- 网络或 AI 错误状态。
- 删除确认状态。
- 未保存离开状态。

Toast 用于轻量成功反馈；需要用户决策或可能造成数据丢失的场景使用对话框或底部抽屉。

## 8. 视觉方向

### 8.1 保留

- 温暖、家庭感的粉色和暖白色基调。
- 手机优先布局。
- 清晰的底部导航。
- 图片作为菜谱的重要视觉信息。
- 轻量成功动效。

### 8.2 调优

- 降低全页面毛玻璃使用量，强化内容层级和可读性。
- 卡片圆角控制在 8px 左右，避免所有元素都呈胶囊形。
- 主要按钮保留珊瑚色，筛选和工具操作使用中性色。
- 做菜详情减少装饰，提升长文本、食材用量和步骤的扫描效率。
- 图标按钮使用统一图标体系，并提供可访问名称。
- 所有页面适配 375px 至 430px 手机宽度。

## 10. 下一版优先级

### P0

- 菜谱确认页可编辑食材和步骤。
- 菜谱详情页可编辑。
- 分类页形成完整浏览路径。
- 复盘历史可查看。
- 导入、保存和删除错误状态完整。

### P1

- 组合筛选和排序。
- 图片设为封面。
- 做菜模式和步骤勾选。
- 复盘时间、备注和快捷改进标签。

### P2

- 手动新建菜谱。
- 导入历史。
- 收藏或常做标记。
- 数据导出与备份。

## 11. 验收标准

1. 用户可以从分享文本完成导入、编辑和保存。
2. 抓取失败时用户输入不丢失，并能通过补充正文继续。
3. 用户能修改 AI 生成的全部核心菜谱字段。
4. 用户能通过搜索、分类和筛选找到菜谱。
5. 菜谱详情在手机上能清晰支持备料和逐步做菜。
6. 用户能记录一次复盘，并在历史中再次找到。
7. 删除、离开未保存页面等高风险操作都有确认。
8. 375px 宽度下无横向滚动、文字遮挡或按钮溢出。
